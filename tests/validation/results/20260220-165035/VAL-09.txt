## Agent Specifications

### Agent 1: SecurityTriageAgent

| Attribute | Detail |
|-----------|--------|
| **Role** | First responder — ingests, classifies, deduplicates, and routes alerts |
| **Framework** | LangGraph `StateGraph` with conditional routing |
| **Trigger** | Polling loop (30s) or webhook from Prometheus Alertmanager |
| **Output** | Incident object with severity, affected service, initial context |

**Tool Assignments:**

| Tool | Purpose | Call Pattern |
|------|---------|:------------:|
| `prometheus_alerts` | Fetch all firing/pending alerts | Every cycle |
| `loki_search_errors` | Search error logs for correlated failures | On alert match |
| `incident_list` | Check for existing open incidents (dedup) | Before create |
| `incident_create` | Create new incident with classification | On new incident |
| `incident_escalate` | Upgrade severity if scope expands | On re-evaluation |

**Classification Decision Tree:**

```python
def classify_severity(alert: Alert) -> str:
    # P1 — Critical: Immediate human notification
    if alert.involves_data_exposure:
        return "sev1"
    if alert.type == "auth_bypass" or alert.type == "credential_leak":
        return "sev1"
    if alert.affected_user_pct > 50:
        return "sev1"
    
    # P2 — Major: Auto-investigate, auto-remediate
    if alert.service_degradation and alert.is_production:
        return "sev2"
    if alert.error_rate > 10:  # >10% error rate
        return "sev2"
    
    # P3 — Minor: Auto-investigate, auto-remediate
    if alert.is_production and alert.limited_scope:
        return "sev3"
    if alert.is_staging:
        return "sev3"
    
    # P4 — Low: Auto-close if known pattern
    return "sev4"
```


### Agent 2: ForensicsAgent

| Attribute | Detail |
|-----------|--------|
| **Role** | Deep investigator — correlates data, builds timeline, identifies root cause |
| **Framework** | LangGraph `StateGraph` with iterative investigation loop |
| **Trigger** | Incident handoff from TriageAgent |
| **Output** | Investigation report with root cause, blast radius, remediation recommendation |

**Tool Assignments:**

| Tool | Purpose | Call Pattern |
|------|---------|:------------:|
| `loki_search_errors` | Deep log search across namespaces/time windows | Multiple passes |
| `k8s_cluster_health` | Check node/pod health for affected services | Initial assessment |
| `k8s_list_deployments` | Map deployment topology of affected service | On investigation start |
| `k8s_get_deployment` | Get detailed config of suspect deployments | Per suspect |
| `incident_get` | Retrieve full incident context from Triage | On handoff |
| `incident_update_status` | Progress to `investigating` → `identified` | On findings |
| `web_search` | Search for CVEs, known attack patterns | If attack suspected |

**Investigation Loop:**

```python
class ForensicsState(TypedDict):
    incident_id: str
    incident_data: dict
    log_evidence: list[dict]         # Timestamped log entries
    k8s_state: dict                  # Cluster/deployment state
    timeline: list[dict]             # Ordered event timeline
    blast_radius: dict               # {services: [], users_affected: int, data_exposed: bool}
    root_cause: str | None           # Identified cause
    classification: Literal["attack", "misconfiguration", "system_failure", "unknown"]
    remediation_recommendation: str  # Suggested fix
    investigation_depth: int         # Loop counter (max 5)

# Investigation proceeds in expanding circles:
# 1. Affected service logs (±15 min window)
# 2. Dependent service logs (upstream/downstream)
# 3. Kubernetes state (pods, deployments, nodes)
# 4. External threat intelligence (CVE search)
# 5. Synthesize timeline + root cause
```

### Agent 3: RemediationAgent

| Attribute | Detail |
|-----------|--------|
| **Role** | Executor — matches runbooks, remediates, verifies, resolves |
| **Framework** | LangGraph `StateGraph` with human-in-the-loop interrupt |
| **Trigger** | Investigation report from ForensicsAgent |
| **Output** | Resolved incident with full action documentation |

**Tool Assignments:**

| Tool | Purpose | Call Pattern |
|------|---------|:------------:|
| `k8s_restart_deployment` | Restart failing pods/deployments | Per runbook step |
| `k8s_scale_deployment` | Scale up/down for capacity issues | Per runbook step |
| `k8s_cluster_health` | Verify health post-remediation | After each action |
| `incident_link_runbook` | Document which runbook was executed | Per runbook |
| `incident_update_status` | Progress to `mitigating` → `resolved` | On status change |
| `incident_resolve` | Close incident with resolution details | On success |
| `incident_escalate` | Escalate if remediation fails | On failure |

**Runbook Matching Matrix:**

| Root Cause Classification | Runbook | Auto-Execute? |
|--------------------------|---------|:-------------:|
| **Pod CrashLoopBackOff** | `restart_deployment` | P2–P4: ✅ / P1: ⏸️ |
| **Resource exhaustion (OOM)** | `scale_deployment` (+50% replicas) | P2–P4: ✅ / P1: ⏸️ |
| **Database connection pool** | `restart_deployment` + `scale_deployment` | P2–P4: ✅ / P1: ⏸️ |
| **FK constraint violation** | `restart_deployment` (clear bad state) | P2–P4: ✅ / P1: ⏸️ |
| **Active attack / data breach** | `isolate_network` + `rotate_credentials` | Always: ⏸️ Human Required |
| **Configuration drift** | `rollback_deployment` | P2–P4: ✅ / P1: ⏸️ |
| **Unknown** | Escalate to human | Always: ⏸️ |

**Approval Gate Logic:**

```python
def requires_human_approval(state: RemediationState) -> bool:
    if state["severity"] == "sev1":
        return True  # Always gate P1
    if state["classification"] == "attack":
        return True  # Always gate active attacks
    if state["blast_radius"]["data_exposed"]:
        return True  # Always gate data exposure
    if state["remediation_action"] in ["isolate_network", "rotate_credentials"]:
        return True  # Destructive actions always gated
    return False
```

```python
    # 4. Verify: k8s_cluster_health
    # 5. Resolve: incident_resolve or escalate
    ...

def health_verification(state: IncidentResponseState) -> dict:

    # Check if affected service recovered
    ...

# ─── Routing Functions ─────────────────────────────────────
def route_after_triage(state) -> str:
    if state["severity"] == "sev4":
        return "auto_close"
    return "forensics"

def route_after_forensics(state) -> str:
    if state["severity"] == "sev1" or state["classification"] == "attack":
        return "approval_gate"
    return "remediation"

def route_after_health_check(state) -> str:
    if state["health_check_passed"]:
        return "resolve"
    if state["retry_count"] < 1:
        return "remediation"  # Retry once
    return "escalate"

# Add nodes
workflow.add_node("triage", triage_agent)
workflow.add_node("forensics", forensics_agent)
workflow.add_node("approval_gate", approval_gate)
workflow.add_node("remediation", remediation_agent)
workflow.add_node("health_check", health_verification)
workflow.add_node("auto_close", lambda s: {"final_status": "resolved"})
workflow.add_node("resolve", lambda s: {"final_status": "resolved"})
workflow.add_node("escalate", lambda s: {"final_status": "escalated"})

# Set entry
workflow.set_entry_point("triage")

workflow.add_conditional_edges("triage", route_after_triage,

workflow.add_conditional_edges("forensics", route_after_forensics,

workflow.add_edge("resolve", END)
```
