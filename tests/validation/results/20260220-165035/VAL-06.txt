### What Failed (Flowise API Proxied Calls)

Every call that requires the MCP proxy to forward an **OBO (On-Behalf-Of) authentication token** to the Flowise backend at `openagentic-api:8000` failed with:

```json
{
  "error": "Unauthorized",
  "message": "No authentication token provided"
}
```

**Affected operations:** `list_chatflows`, `list_nodes`, `create_chatflow`, `create_from_template`, `create_chatflow_advanced`

### Contradiction: Health Check vs Framework Status

| Source | Flowise Status | Endpoint |
|--------|:--------------:|----------|
| `flowise_health_check` | ✅ Healthy | Local ping |
| `get_framework_status` | ❌ Unhealthy | `http://openagentic-api:8000/api/flowise-workspace` |

The health check is **misleading** — it reports healthy based on a local/shallow check, while the actual API workspace endpoint is unreachable or rejecting tokens.

## Recommendations

| Priority | Action | Rationale |
|:--------:|--------|-----------|
| **P0** | **Fix OBO token forwarding** in the MCP proxy → Flowise API path. The token is present (auth_status shows `userAccessToken` exists) but is not being attached to outbound Flowise requests. | **All CRUD operations are blocked.** Zero chatflows can be managed. |
| **P0** | **Fix the `flowise-workspace` endpoint** at `openagentic-api:8000`. The `get_framework_status` probe confirms it's unhealthy. | Endpoint may be down independently of the auth issue. |
| **P1** | **Fix `flowise_health_check` to probe the real API**, not just return a local status. It currently gives a false-positive "healthy" while the actual backend is down. | Misleading health signal undermines trust in monitoring. |
| **P1** | **Fix `flowise_get_auth_status` Pydantic validation error** — the `meta` parameter injection is causing a `unexpected_keyword_argument` crash. | Auth debugging tool is itself broken. |
| **P2** | **Add circuit-breaker logic** — after N consecutive 401s, fail-fast with a clear diagnostic message instead of silently returning 401 on every call. | Improves operator experience during outages. |
