## Root Cause Analysis

### New Error vs. Previous Outages

| Audit | Error Signature | Proxy State |
|:-----:|-----------------|:-----------:|
| Audits 1–3 | `ECONNREFUSED 10.43.224.106:8080` | Pod down / crashed |

| **Audit 5 (now)** | **`HTTP 500: Attempt to overwrite 'args' in LogRecord`** | **Running but broken** |

### Diagnosis

The error `"Attempt to overwrite 'args' in LogRecord"` is a **known Python `logging` module issue** that occurs when:

1. **A custom log formatter or handler** attempts to set `args` on a `LogRecord` object that already has it populated — this is disallowed in Python 3.12+
2. **Concurrent log writes** from multiple async tool handlers create a race condition on shared `LogRecord` instances
3. **A library upgrade** (e.g., `structlog`, `python-json-logger`, or the `logging` stdlib itself) introduced stricter immutability checks

> [!IMPORTANT]
> The proxy **is running** (it responds with HTTP 500 instead of ECONNREFUSED), but it crashes on **every request** before reaching any cloud API. This is a code-level bug, not an infrastructure outage.
