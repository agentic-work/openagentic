## Key Findings

> [!WARNING]
> **Primary contributor to latency:** The PostgreSQL FK constraint violations on `user_query_audit` are occurring on every tool call, creating cascading retry/failure cycles. This is the **same issue** identified in the previous incident `INC-20260220-0001`.

**1. PostgreSQL Audit Race Condition (P0)**
- **Pattern:** `INSERT INTO user_query_audit` fires before the parent `message` record commits
- **Rate:** ~174 errors in 15 minutes (~11.6/min)
- **Impact:** Audit trail is silently broken; each failed INSERT adds latency to the request path

**2. Two OpenAgentic Pods DOWN (P1)**
- **code-manager** (`10.42.5.19:8080`) and **openagentic-synth** (`10.42.2.37:8090`) are unreachable
- **Impact:** Code execution and OAT workflows are offline

**3. Milvus Schema Mismatches (P2)**
- Embedding dimension mismatch (1024 vs 1536) blocking conversation memory writes
- VARCHAR overflow in `tool_success_tracking` collection
- Missing collections and fields suggesting schema drift
