### Root Cause Analysis

The error `Attempt to overwrite 'args' in LogRecord` is a **known Python bug** in the `logging` module. It occurs when:

1. **A custom log handler or formatter** attempts to set the `args` attribute on a `LogRecord` that already has one — typically a **thread-safety race condition** in a multi-threaded async server.
2. **A recent deployment or dependency upgrade** introduced a logging middleware that conflicts with the existing `LogRecord` factory.

This is **not an infrastructure failure** (PostgreSQL/Redis/Milvus are likely running fine). The admin MCP **Python process itself crashes** before it can reach any backend, which is why the error is identical across all five different database probes.
