What this is
Three full-page mocks showing what codemode will render after the 10-phase TUI-parity port lands. Built from the gap analysis between openagentic (Ink CLI) and the current React UI. Every visual primitive in each mock corresponds to a numbered gap in the design doc — these double as acceptance criteria.
MOCK 1 · DEPLOY DEBUG
3 turns, ~17 min, AKS+CSI-S3 hash divergence. Demonstrates: parallel reads & bash, both-side edit diff, sub-agent dispatch, todoWrite checkboxes, model-swap boundary, compacted boundary, slash palette mid-flow, status-line.
MOCK 2 · FULLSTACK BUILD
3 turns, ~34 min, Python+React URL shortener from scratch. Demonstrates: skill-invoked + plugin-loaded boundaries, 12-tool parallel writes, npm install with stall-detect, test-runner sub-agent, artifact panels (README + issue), live preview iframe, custom error handler.
MOCK 3 · RESEARCH-FIX-DEPLOY
3 turns, ~1h 17m, Ollama "Stream completed outputTokens=0" root cause + TDD fix. Demonstrates: parallel grep+read+webfetch, multi-stage thinking blocks with rainbow keywords, 2 sub-agents, both-side diff with word-level highlights, red→green TDD cycle, build+helm+rollout, Playwright verification.

How to use these
  1. Click a mock — it loads as a single-page HTML file using only the --cm-* tokens that ship today.
  2. Each visual primitive corresponds to a numbered gap in docs/superpowers/specs/2026-04-25-codemode-tui-parity-design.md.
  3. Per the spec, each phase is TDD: render the React component, snapshot the DOM, assert it matches the mock byte-for-byte (modulo content).
  4. Three themes are configured in the codebase (catppuccin-mocha shown here, dracula, terminal-green); all three need snapshot pass.
