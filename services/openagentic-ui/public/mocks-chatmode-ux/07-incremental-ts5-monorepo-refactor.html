Thought for 5.6s · 812 tokens

A 40-file monorepo refactor to TS5 strict has three failure modes I want to design around:

  • Big-bang conversions fail. If I try to convert everything in one go the TypeScript errors cascade and the agent can't make progress. Instead I'll do a wave migration: leaves of the dep graph first, then upstream, so each service's deps are already typed by the time it's its turn.
  • Strict mode finds bugs. That's actually good, but it means I can't just flip "strict": true and hope — every service will produce 5-50 new errors that I need to fix. I'll budget for this and let testing-agent verify coverage doesn't drop.
  • Project references are pickier than tsc-standalone. Circular imports that TS2 tolerated will break. I'll have codegen-agent run madge --circular first and break cycles before the migration.

Plan: dep-graph the repo, delegate codegen to 4 sub-agents (codegen, security, testing, docgen), do pass 1, expect failures, do pass 2 targeted patch.

Tests must not regress — that's the user's hard constraint. If coverage drops I'll abort and report.

What I changed in tsconfig

  • Root tsconfig.base.json centralises strict: true, noUncheckedIndexedAccess: true, exactOptionalPropertyTypes: true, moduleResolution: "bundler".
  • Each package gets a minimal tsconfig.build.json with composite: true and an explicit references array. Total config surface dropped from 11 unique tsconfigs to 1 base + 13 projects.
  • Removed allowJs — no more JS fallback. Any new file must be .ts or the build breaks.

Risk notes

  • The auth middleware narrow (security finding #1) changed req.user's type from implicit any to User | undefined. Existing callers that didn't check nullability now get a type error at callsite. I fixed those in pass 2 but if any dynamic plugin loads middleware at runtime they'll break until they update their own types.
  • packages/shared-logger/format.ts broke a circular import by extracting a tiny level.ts helper. The exported shape is unchanged, but if anyone imported “shared-logger/format” by deep path they now pull a smaller bundle. Should be a non-event, but worth a grep.
  • exactOptionalPropertyTypes is the picky one — it rejects { foo: undefined } where foo is declared foo?: string. Four locations were quietly relying on this; I fixed them but any future PRs that forget will fail CI.
